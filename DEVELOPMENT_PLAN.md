## 五、技术债务与架构决策

### 5.1 当前技术债务

| 债务 | 影响 | 状态 |
|---|---|---|
| SQLite 单文件 | 并发限制 | 已启用 WAL 模式 |
| 无用户认证 | 任何人可访问 | ✅ 已支持 `LOGPAD_ACCESS_TOKEN` / `LOGPAD_PASSWORD` |
| API 错误处理 | 可能崩溃 | ✅ 已统一加 try/catch |
| 硬编码路径 | 换机器需改代码 | ✅ 已提取到环境变量 |
| 无类型安全 | 运行时错误 | ✅ 已补全接口类型定义和 typecheck |

### 5.2 架构决策记录

**决策 1：本地优先**
- 原因：素材大、隐私敏感、不依赖网络
- 实现：Next.js 在 Mac mini 本地跑，SQLite 存本地
- 风险：备份需要自己处理

**决策 2：文件系统 + 数据库双存储**
- 原因：`runs/` 目录结构是现有系统的核心，不能改
- 实现：文件系统存完整内容，SQLite 存索引和状态
- 风险：需要维护一致性

**决策 3：增量集成现有系统**
- 原因：不替换 `media_pipeline.py` 和文件夹结构
- 实现：Web UI 只是调用层和展示层
- 风险：两套系统并行，需要明确边界
